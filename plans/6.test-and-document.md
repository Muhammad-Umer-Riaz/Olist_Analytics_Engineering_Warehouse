## Task 1 — Singular tests (new files in `dbt/tests/`)

Custom `SELECT`s that **return the offending rows** (0 rows = pass). One concern per
file; reuse the canonical definitions already in the models so tests stay consistent
with the build.

1. **`assert_no_negative_money.sql`** — `fct_order_items` where any of
   `price_brl, freight_brl, revenue_brl, revenue_usd, revenue_eur < 0`.
2. **`assert_delivered_after_purchase.sql`** — `fct_orders` where
   `order_delivered_customer_date < order_purchase_timestamp`, only where delivered
   is not null (3-state).
3. **`assert_delivered_orders_have_delivery_date.sql`** — conditional-null per Q6:
   `order_status = 'delivered'` and `order_delivered_customer_date_key is null`.
   Forward direction only, as an **error**. Reverse not asserted (canceled/other
   statuses can legitimately carry a delivery timestamp).
4. **`assert_fct_items_reconcile_to_orders.sql`** — cross-fact reconciliation: group
   `fct_order_items.revenue_brl` by `order_id`, join `fct_orders.order_item_total`,
   return any order where `abs(diff) > 0.01`.

**Not added (already covered):** orphan fact keys (the fact-FK `relationships` tests
already do this); "payment reconciles" as a gate (the 3-state `is_payment_reconciled
= FALSE` rows are legitimate financing fees, kept by design — ADR-009/015).
