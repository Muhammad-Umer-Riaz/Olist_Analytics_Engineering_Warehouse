# Phase 1 — Snowflake Warehouse Setup (L2)

**Complexity:** ⚠️ Medium — owner runs DDL in Snowsight; Claude generates SQL + keys + verifies.
**Status:** In progress · _Started 2026-06-16_

## Context

Phase 0 produced a working local toolchain (dlt + dbt on Python 3.12) and a confirmed
Snowflake account (`NEB29791`, AWS, ACCOUNTADMIN; 400 trial credits, 109 days left).
Phase 1 stands up the Snowflake objects everything depends on: a warehouse, the
`OLIST` database, the four schemas, a cost backstop, and **least-privilege RBAC** so
dlt and dbt connect as separate, scoped service accounts.

Owner drives the Snowflake UI; Claude generates SQL and guides (first time with Snowflake).

## Decisions (settled via grilling)

1. **Two service users / two roles / two key-pairs** — enforced separation:
   - `OLIST_LOADER` → `OLIST_LOADER_SVC` (dlt): writes `RAW`
   - `OLIST_TRANSFORMER` → `OLIST_TRANSFORMER_SVC` (dbt): reads `RAW`, writes `STAGING`/`INTERMEDIATE`/`MARTS`
2. **Ownership = Option C:** `SYSADMIN` owns warehouse + database; custom roles get
   privilege grants (not schema ownership); tables are owned by their creating role.
3. **No managed-access schemas;** cross-role read via ACCOUNTADMIN/SECURITYADMIN
   future grant (`SELECT ON FUTURE TABLES IN SCHEMA RAW` → TRANSFORMER).
4. **Role-switching:** `SYSADMIN` (infra), `SECURITYADMIN` (roles/users/grants),
   `ACCOUNTADMIN` (resource monitor).
5. **Auth:** key-pair on `TYPE = SERVICE` users; **unencrypted** PKCS#8 keys in
   `.keys/` (gitignored; committed README placeholder).
6. **Loader also gets `CREATE SCHEMA ON DATABASE OLIST`** for dlt's `RAW_STAGING` merge scratch.
7. **Cost:** `OLIST_WH_MONITOR`, 30 credits/month (notify 75/90, SUSPEND 100, SUSPEND_IMMEDIATE 110).
