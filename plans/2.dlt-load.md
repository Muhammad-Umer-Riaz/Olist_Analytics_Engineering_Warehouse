# Phase 2 — Load Layer: dlt → Snowflake RAW (L1)

**Complexity:** ⚠️ Medium — first dlt pipeline; owner drove the runs, Claude generated + guided.
**Status:** ✅ Complete · _2026-06-16_ (design grilled; see `DECISIONS.md` ADR-013).

## Context

Phase 1 stood up Snowflake (warehouse, `OLIST` db, the four schemas, least-privilege
RBAC) and verified the `OLIST_LOADER_SVC` key-pair can write `RAW`. Phase 2 built the
**EL** of ELT: a dlt pipeline that lands the 9 Olist CSVs **plus** FX rates into
`OLIST.RAW`, faithfully and idempotently, so dbt (Phase 3+) has a trustworthy source.
dlt is the loader (ADR-003); RAW is a 1:1 landing zone; loads are idempotent.

## Decisions (resolved via grilling; full rationale in ADR-013)

1. `customers` → **full-refresh** → 5 full-refresh + 4 merge.
2. **RAW stays 1:1.** Cursor incremental only on `orders` + `order_reviews`;
   `order_items` + `order_payments` merge on composite PK (no cursor). No derived columns.
3. Two-pass seed: pass 1 = through 2017, pass 2 = 2018. `orders`/`reviews` split on their
   own cursor; `order_items`/`order_payments` on parent-order-id membership.
4. Merge keys: orders `order_id`; order_items `(order_id, order_item_id)`;
   order_payments `(order_id, payment_sequential)`; order_reviews `(review_id, order_id)`.
5. Extraction = pandas `@dlt.resource` per table, `dtype=str` on zip prefixes.
6. FX = Frankfurter long format over 2016-09-04…2018-10-17, BRL→USD/EUR.
7. Self-contained dlt project in `dlt/`; key-pair auth via `private_key_path`;
   `dataset_name="raw"` → `OLIST.RAW`.
