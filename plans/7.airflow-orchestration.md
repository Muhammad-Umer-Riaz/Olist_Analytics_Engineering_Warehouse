## Goal
Wire the existing dlt load + dbt transform into one Airflow DAG with real dependencies,
retries, and a failure branch. **dlt stays the loader; Airflow only orchestrates (ADR-003).**

## Key forks (owner-confirmed via grill)
- dbt in Airflow ‚Üí **astronomer-cosmos** (model-level tasks), not a BashOperator.
- Credentials ‚Üí **two Airflow Connections** (loader + transformer), least privilege.
- Load side ‚Üí **one `dlt_load_olist` task** running `load_olist.py` unchanged.

## The central challenge
Astro runs Airflow in **Docker containers**; the working setup relies on host absolute
paths (`.keys/*.p8`, `data/raw/`) that don't exist in a container. The phase is mostly
about bridging host ‚Üí container (mounts + Connections + isolated venvs).
