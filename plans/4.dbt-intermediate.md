## Confirmed decisions (owner, this session)

1. **Reconciliation** = `sum(payment_value)` vs `sum(price + freight)` at **order grain**,
   over orders having **both** sides. Tolerance **Â±0.01 BRL absolute**.
2. **Quarantine vs flag:** reconciliation discrepancies are **flagged, not quarantined**.
   `is_payment_reconciled` is **3-state** (`true`/`false`/**`null`** = not assessable) plus
   signed `payment_reconciliation_diff`. Orders kept in facts; audit = `where not
   is_payment_reconciled`. Rejects table holds only genuinely-broken/excluded rows.
3. **FX gap-fill:** date_spine over **full extent** `least(min order, min rate) â†’ greatest(
   max order, max rate)`; **LOCF forward-fill + leading back-fill**; **fail-loud** coverage
   check (zero null rates over order range). Convert on **`order_purchase_timestamp::date`**.
4. **Payment type:** `primary_payment_type` = argmax `sum(payment_value)`, **tie-break
   alphabetical** (deterministic); + totals/count/max_installments/distinct_types/is_multi_method.
5. **Review dedup:** keep **latest** by `review_creation_date` (tie-break `review_answer_timestamp`,
   then `review_id`). `review_id` stays non-unique across orders; test **order_id** unique.
6. **Category PTâ†’EN** in `int_olist__products_enriched`; untranslated = kept null (signal).
7. **Materialization** = `view`; **naming** = `int_olist__<entity>_<verb>`.
8. **Macros built where first used.** Phase 4: `delivery_days`, `is_late`, `order_item_revenue`,
   `brl_to`. `rfm_bucket`/AOV **deferred to Phase 5** (person/mart grain). Item-grain FX **deferred
   to Phase 5** (reuses `brl_to` + `int_olist__fx_rates_filled`).
