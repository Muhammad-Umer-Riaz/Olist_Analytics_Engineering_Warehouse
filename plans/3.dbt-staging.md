# Phase 3 — Transform: dbt Staging (L3)

**Complexity:** ⚠️ Medium — first dbt project setup + key-pair connection + 10 staging models.
**Status:** ✅ Complete · _2026-06-17_ (plan grilled via `/grill-me`; full rationale in `DECISIONS.md` ADR-014).

## Context

Phases 0–2 stood up Snowflake (RBAC, ADR-012) and landed all 9 Olist tables + `fx_rates`
into `OLIST.RAW` faithfully 1:1 (timestamps as VARCHAR, zip prefixes as strings). Phase 3
built the **first dbt layer**: `stg_olist__*` views that cast + rename RAW into a typed,
trustworthy 1:1 contract for the intermediate/marts layers, and established the dbt project
conventions (naming, schema routing, tests) for all of L3.

## Decisions (resolved via grilling; full rationale in ADR-014)

1. **Scaffold** manually into existing `dbt/` (no interactive `dbt init`); owner runs every
   command in PowerShell via the venv.
2. **Connection:** `profiles.yml` in `dbt/` (gitignored), key-pair as `OLIST_TRANSFORMER_SVC`,
   run with `--profiles-dir .`. dbt Power User extension pointed at the same dir.
3. **Naming (dbt-Labs):** `stg_olist__<entity>`, `_olist__sources.yml`, `_olist__models.yml`;
   source `olist_raw` over `OLIST.RAW`.
4. **Schema routing:** `generate_schema_name` override so `+schema` is used verbatim (avoids
   `STAGING_STAGING`); no CREATE SCHEMA pre-grant (schemas pre-exist).
5. **Typing:** staging owns casting; hard `::timestamp_ntz` (fail-fast), NTZ (local Brazil,
   no offset); drop all `_dlt_*` (lineage stays in RAW).
6. **Renames:** light-touch — fix `product_*_lenght`→`length`; keep zip names entity-specific;
   otherwise mirror RAW.
7. **Geolocation:** collapse to 1 row/zip via `MEDIAN` coords + deterministic ranked modal
   city/state (not bare `MODE()`); keep coord-NULL zips.
8. **Category translation:** `stg_olist__products` stays 1:1; translation its own model; PT→EN
   join deferred to intermediate/marts (refines `CONTEXT.md §4`).
9. **`dbt_utils`** added for composite-PK tests (and later surrogate keys).
10. **Tests (moderate):** PK unique + not_null on every model, accepted_values on
    `order_status`/`payment_type`/`review_score`; generic-test args under `arguments:`.

## What was built

- `dbt/dbt_project.yml`, `dbt/profiles.yml` (gitignored), `dbt/packages.yml` (`dbt_utils 1.3.3`),
  `dbt/macros/generate_schema_name.sql`.
- `dbt/models/staging/olist/_olist__sources.yml` — source `olist_raw`, 10 tables.
- 10 `stg_olist__*.sql` views (customers, sellers, products, category_translation,
  geolocation, orders, order_items, order_payments, order_reviews, fx_rates).
- `dbt/models/staging/olist/_olist__models.yml` — 32 tests + descriptions.
- `.gitignore` fix: the `profiles.yml` / `*.p8` lines had **inline comments**, which `.gitignore`
  treats as part of the pattern — so `profiles.yml` was NOT actually being ignored. Moved the
  comments to their own lines (same gotcha noted in Phase 2).

## Verification (passed)

- `dbt deps` → `dbt_utils 1.3.3`; `dbt parse` clean.
- `dbt debug --profiles-dir .` → **All checks passed** (transformer key-pair → OLIST, sees STAGING).
- `dbt run --select staging` → **10 views** built in `OLIST.STAGING` (not `STAGING_STAGING` — macro works).
- Row-count reconciliation vs RAW: all 9 match exactly; `geolocation` collapsed
  **1,000,163 → 19,015** (distinct zip prefixes); `category_translation` = 71.
- `dbt test --select staging` → **PASS=32, ERROR=0**, no deprecations after nesting test args.
